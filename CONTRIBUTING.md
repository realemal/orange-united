# Contributing to Orange United

Thank you for your interest in contributing to Orange United! This document provides guidelines for contributing to the project.

## Getting Started

1. Fork the repository
2. Clone your fork locally
3. Install dependencies: `npm install`
4. Create a new branch: `git checkout -b feature/your-feature-name`
5. Make your changes
6. Test locally: `npm run dev`
7. Build to verify: `npm run build`
8. Commit your changes: `git commit -m "Description of changes"`
9. Push to your fork: `git push origin feature/your-feature-name`
10. Open a pull request

## Code Style

- Use TypeScript where possible
- Follow existing code formatting (we use Prettier-compatible style)
- Write semantic, accessible HTML
- Use Tailwind CSS utility classes for styling
- Add comments for complex logic

## Content Guidelines

### Articles

- Keep titles clear and descriptive
- Provide accurate excerpts (1-2 sentences)
- Use proper categories (Curriculum, Board Meetings, Opinion, News)
- Add relevant tags
- Include Spanish translations when possible
- Cite sources for factual claims

### Events

- Include complete date, time, and location
- Provide clear descriptions
- Add RSVP links when available
- Mark events as canceled promptly if needed

## Commit Messages

Use clear, descriptive commit messages:

- `feat: Add new component for event filtering`
- `fix: Resolve mobile menu toggle issue`
- `docs: Update README with deployment instructions`
- `style: Improve article card hover effects`
- `refactor: Simplify newsletter form logic`

## Testing

Before submitting a pull request:

1. Test locally in development mode
2. Build and preview production build
3. Check responsive design (mobile, tablet, desktop)
4. Verify accessibility (keyboard navigation, focus states)
5. Test forms (newsletter, contact, volunteer)
6. Check all links work

## Pull Request Process

1. Update documentation if needed
2. Add screenshots for UI changes
3. Describe what your PR does and why
4. Reference any related issues
5. Wait for review and address feedback

## Code of Conduct

- Be respectful and constructive
- Focus on the content, not the person
- Assume good intentions
- Help others learn and grow

## Questions?

If you have questions about contributing, please:
- Open an issue for discussion
- Reach out via the contact form
- Ask in pull request comments
